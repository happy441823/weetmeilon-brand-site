# MAINTENANCE.md

## 每周检查事项

- 首页是否正常打开。
- 天猫旗舰店与京东旗舰店跳转按钮是否仍指向正确链接。
- 年龄提示是否正常出现。
- 产品页、FAQ、隐私发货和清洁指南是否可访问。
- sitemap 和 robots 是否可访问。
- 统计事件是否正常上报。
- 是否有 404 或页面加载异常。

## 每月 SEO 内容计划

- 新增或扩写 4 篇文章。
- 检查品牌词、材质词、隐私发货、清洁保养相关搜索表现。
- 将高点击低转化页面补充 CTA 或 FAQ。
- 将高跳出文章增加产品系列入口和官方渠道按钮。

## 产品上新流程

1. 在 `src/lib/products.ts` 新增产品对象。
2. 将产品图放入 `public/images/products/`，并在 `src/lib/products.ts` 维护对应图片路径。
3. 检查 `products/[slug]` 自动生成详情页。
4. 首页如需重点露出，调整 `src/app/page.tsx` 的展示顺序或新增模块。
5. 新增一篇新品介绍文章到 `src/lib/articles.ts`。
6. 自测产品页、官方渠道按钮、移动端展示和 sitemap。

## 首页 Banner 更新流程

1. 替换首页产品图、品牌细节图或新增真实主视觉素材。
2. 检查图片是否克制、合规、无露骨表达。
3. 调整首页 H1、说明和 CTA。
4. 移动端检查首屏是否能看到主标题和官方渠道购买按钮。

## 活动页更新流程

1. 确认活动信息来自天猫店铺真实页面。
2. 不写虚假折扣、库存、销量或倒计时。
3. 新增活动模块时使用统一 `StoreButtons`。
4. 活动结束后及时下线或更新。

## 数据异常检查

- UV 突降：检查部署、域名、统计脚本、搜索收录。
- 官方渠道点击率低：检查按钮位置、文案、首屏信息和页面速度。
- FAQ 浏览高但点击低：补充产品选择入口和购买路径说明。
- 某渠道跳出高：检查落地页是否匹配投放文案。

## 官方渠道链接检查

- 上线前将 `NEXT_PUBLIC_TMALL_STORE_URL` 和 `NEXT_PUBLIC_JD_STORE_URL` 替换为真实链接。
- 每周点击首页、产品页、FAQ、文章页和页脚按钮。
- 活动期检查优惠链接是否过期。

## 合规文案检查

- 不面向未成年人。
- 不写医疗、健康、情感或能力效果承诺。
- 不虚构安全认证、检测报告、专利、销量或用户评价。
- 图片不低俗、不露骨，不过度暗示。
- 原生肌凝硅只作为品牌材质科技表达。

## 备份与版本管理

- 使用 Git 管理代码。
- 每次上线前创建 tag 或 release。
- 产品图和文案更新保留原始素材备份。
- 重大活动上线前保留可回滚版本。

## 后续迭代建议

- 接入真实产品图和天猫、京东真实链接。
- 增加活动落地页模板。
- 增加渠道专属落地页，例如抖音版、小红书版、百度搜索版。
- 增加 FAQ 结构化数据。
- 增加文章正文编辑流程或 CMS。
- 增加官方渠道跳转参数，辅助判断渠道贡献。
